# Workflow Implementation

## Overview

Implement AI-powered workflow systems with agent orchestration, task sequencing, and human-AI collaboration. Use detailed procedures for comprehensive implementation guidance.

**Announce at start:** "I'm using the workflow-implementation skill to implement this AI-powered workflow system."

## When to Use This Skill

**Trigger Conditions:**
- Need to implement workflow automation with AI integration (4-8 prompts)
- Building agent-centric workflow pages requiring enterprise integrations
- Setting up human-in-the-loop (HITL) workflow management
- Creating multi-discipline collaboration frameworks
- Developing workflow automation with intelligent task sequencing

**Prerequisites:**
- Clear workflow requirements defined
- Target discipline(s) identified
- Access to workflow implementation procedures

## Step-by-Step Procedure

### Step 1: Assess Requirements
- Analyze existing workflow processes
- Identify automation opportunities
- Define success criteria and constraints

### Step 2: Select Implementation Approach
- Choose between standard workflow (0001) or agent-centric (0002) procedures
- Evaluate complexity and integration requirements
- Determine HITL and multi-discipline needs

### Step 3: Execute Implementation
- Follow selected procedure's phased approach
- Implement agent orchestration and prompt management
- Set up user interfaces and testing frameworks

### Step 4: Deploy and Monitor
- Use feature flags for controlled rollout
- Monitor performance and user adoption
- Implement continuous optimization

## Success Criteria

- [ ] Workflow requirements clearly defined and documented
- [ ] Appropriate procedure selected based on complexity
- [ ] Implementation follows established patterns and phases
- [ ] Agent orchestration and HITL properly configured
- [ ] Testing and monitoring frameworks operational
- [ ] Successful deployment with performance monitoring

## Common Pitfalls

1. **Wrong Procedure Selection** - Choose 0001 for standard workflows, 0002 for complex agent-centric pages
2. **Insufficient Requirements** - Always complete thorough workflow analysis first
3. **Missing HITL Integration** - Include human oversight for critical decisions
4. **Poor Testing Coverage** - Implement comprehensive testing before deployment
